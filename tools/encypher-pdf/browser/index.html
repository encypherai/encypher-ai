What to look for
  • Normal title and paragraph spacing
  • Readable body typography
  • Conventional heading hierarchy
  • No visible artifacts from invisible provenance markers
  • Reasonable output across sans-serif and serif families
  • These samples are visual and Unicode-fidelity demos
  • They are not enterprise-signed verification fixtures
  • Pasting their extracted text into the Sign/Verify demo will not validate unless the text was produced by the actual signing service
  • The signed fixtures below are produced by the live local signing workflow
  • Their verification reports show verification_status: Success and reason_code: OK
  • Use those fixtures when testing the marketing-site verify flow or raw verify API
  • The enterprise fixture matrix uses sentence-level micro signing from /api/v1/sign
  • Both VS256-RS and legacy_safe_rs now return an embedding_plan when requested
  • The mode matrix covers embed_c2pa=true and embed_c2pa=false for both families
