# Dashboard Diagrams

This directory contains dashboard-specific D2 sources for engineering and customer-facing workflows.

## Diagram Set

- `dashboard-system-context.d2` — how the dashboard sits between users, sessions, org context, APIs, and backend services
- `dashboard-onboarding-flow.d2` — how a new user gets from signup to first successful workflow
- `publisher-cms-workflow.d2` — how a customer goes from dashboard configuration to signed publishing and reader verification
- `enterprise-editorial-review-flow.d2` — how enterprise editorial review and custom assertions map into signing and evidence
- `webhook-lifecycle.d2` — how event delivery reaches customer systems and downstream automations

Prefer linking these diagrams from the README and guide sections they explain.
