# The Encypher Times

A demonstration newspaper where every article, image, audio clip, and video is cryptographically signed using Encypher's enterprise content provenance infrastructure. The publication is fictional. The signatures are real.

Built on the C2PA open standard (Section A.7), this site shows content provenance at publication scale -- 8 articles across 5 sections, each signed at the sentence level with tamper-proof proof of origin.

## Quick Start

```bash
npm install
npm run build        # static export to out/
npx serve out        # serve locally
```

For development:

```bash
npm run dev          # starts on port 3060
```

Note: Dynamic article routes (`/article/[slug]`) require `generateStaticParams` and only work in the static build, not dev mode. Use `npx serve out` for full functionality.

## Environment

Copy `.env.example` and configure:

```bash
cp .env.example .env.local
```

## Seed Script

The seed script signs all articles and media against the Encypher enterprise API:

```bash
cd seed
pip install -r requirements.txt
python seed.py --api-url http://localhost:8000 --api-key your-key
```

This generates `public/signed-content/` (article JSON with embedded ZWC signatures) and signs images/audio/video with C2PA manifests into `public/signed-media/`.

## Walkthrough Screenshots

A Puppeteer script captures 17 screenshots of the site for review artifacts:

```bash
# Build and serve the site first
npm run build
npx serve out -l 3070

# Capture screenshots and create zip
node scripts/capture-walkthrough.js [options]
```

| Flag | Default | Description |
|------|---------|-------------|
| `--url <base>` | `http://localhost:3070` | Base URL of the running site |
| `--out <dir>` | `./walkthrough` | Directory for screenshot PNGs |
| `--zip <path>` | `./encypher-times-walkthrough.zip` | Output zip file path |
| `--skip-states` | | Skip mock verification result screenshots |

The script captures:
- Homepage (hero, article grid, sections, footer)
- Article page (hero with C2PA badge, body, signing box)
- Verify page with mock result states (verified, tampered, how-it-works)
- About page
- Extension install banner
- Mobile views (375px)

Requires Puppeteer: `npm install puppeteer`, or set `NODE_PATH` to a directory with puppeteer installed.

## Static Export

The site builds as a Next.js static export (`output: "export"` in `next.config.ts`). The `out/` directory can be deployed to any static host -- no Node.js server required.

## Key Design Decisions

- **Sentence-level signing**: Every sentence carries invisible ZWC markers with HMAC signatures, not just document-level hashes
- **Three verification states**: Verified (teal), Tampered (amber), No Signature (gray)
- **Capability-accurate copy**: All text explicitly scopes detection to "at the point of ingestion" -- no claims about detecting content in AI-generated outputs
- **Collapsible technical details**: Outcome-focused language for prospect demos, engineer-facing details behind a disclosure toggle
- **C2PA image manifests**: Hero images independently signed, verifiable by any C2PA-compatible tool
