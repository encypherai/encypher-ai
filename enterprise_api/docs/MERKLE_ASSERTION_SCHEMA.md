# Merkle Assertion Contract (`com.encypher.merkle.v1`)

## Purpose

`com.encypher.merkle.v1` is a custom C2PA assertion used to bind signed content to a Merkle root generated by Enterprise API segmentation/indexing flows.

This assertion gives verifiers and downstream systems a stable contract for Merkle linkage metadata independent of internal DB schemas.

## Versioning

- **Assertion label**: `com.encypher.merkle.v1`
- **Contract version**: `v1`
- **Compatibility rule**:
  - Additive fields are backward-compatible.
  - Renames/removals require a new label version (`com.encypher.merkle.v2`).

## Data Schema (v1)

### Required fields

| Field | Type | Description |
|---|---|---|
| `root_hash` | string | Hex-encoded Merkle root hash (SHA-256, 64 chars). |
| `total_segments` | integer | Number of signed segments represented by the root. |

### Optional fields

| Field | Type | Description |
|---|---|---|
| `root_id` | string | UUID of the persisted Merkle root (`merkle_roots.id`) when available. |
| `segmentation_level` | string | Segmentation granularity used to build the root (`sentence`, `paragraph`, `section`, `document`, etc.). |

## Canonical example

```json
{
  "label": "com.encypher.merkle.v1",
  "data": {
    "root_hash": "8f7c4f8d4c3a8ecb798d8f0b55f4e48e96d9b6d9c3bbaf0f5f0f02d2b2d2a1ab",
    "root_id": "2f0e8d3c-9f10-4d8d-b770-f6f8da4c2ae4",
    "segmentation_level": "sentence",
    "total_segments": 42
  }
}
```

## Production semantics

- Assertion is included when Merkle context is available at signing time.
- `root_hash` is the primary integrity linkage field.
- `root_id` is an optimization pointer for DB-backed proof retrieval, not a cryptographic requirement.

## Verification guidance

- Verifiers should always validate `root_hash` format and treat it as canonical.
- If `root_id` is present, systems may fetch proof material from storage.
- Missing optional fields (`root_id`, `segmentation_level`) must not invalidate a signature by themselves.
