## Core framing

The plugin should be presented as a WordPress-native proof that provenance can become an interoperable layer for AI-era publishing.

This is not just a commercial plugin story.

This is a:

- standards-based provenance story
- WordPress-native workflow story
- extensibility and ecosystem story
- future AI publishing infrastructure story

## Meeting goal

The goal is not to ask WordPress core to adopt an Encypher backend.

The goal is to show:

- WordPress needs a provenance/authenticity layer for the AI era
- this plugin demonstrates a credible implementation path
- the architecture can support broader ecosystem participation
- Encypher is proving the workflow, not trying to close the ecosystem around a single vendor path
